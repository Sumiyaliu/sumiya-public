# 📦 百宝箱 HTML 工具输出与部署 SOP (Standard Operating Procedure)

**适用范围**：面向客户的在线课程培训资料、交互式诊断报告、业务推介 OnePage 等 Web 资产的快速生产与上线。
**核心原则**：内容模块化、数据脱敏合规、UI 组件复用、部署标准化。

---

## 一、 👥 角色分工与协作流
根据《百宝箱档案》规范，本流程涉及三个核心角色：
1. **策划 (Planner)**：负责提炼业务逻辑、撰写 Markdown/PPT 提纲，定义核心数据与输出目标（如：融合“四力飞轮”、“场景感知召回”等业务抓手）。
2. **开发 (Developer / AI Assistant)**：负责将文字提纲转化为高交互的 HTML 页面，复用基础 UI 模板，实现数据可视化与排版。
3. **部署 (Deployer)**：负责文件审查、版本迭代及 GitHub Pages 线上环境更新。

---

## 二、 🔄 标准生产四步法

### Step 1: 提纲撰写与结构定义 (Planner)
*   **内容结构**：按 `封面 -> 现状痛点(金字塔大盘) -> 拆解破局点(如唯一性破解/基建动作) -> 自动化工具(智投/油门工作台) -> 总结Action` 推进。
*   **文案要求**：
    *   **结论先行**：标题直接给出现象或结论（如：“基建拉满第一道坎：破解 ADQ 唯一性校验”）。
    *   **业务包装**：将基础操作包装为策略抓手（如：“主动干预人货场”、“增加考核单元要素”）。

### Step 2: 模板复用与 HTML 开发 (Developer)
**强烈建议基于最新的 `食饮投放力诊断与提升_客户培训_v6.html` 作为底层脚手架进行修改**，以复用以下核心资产：
*   **流式网格系统**：使用 TailwindCSS 的 `grid grid-cols-12` 实现左右两栏或多栏的灵活布局。
*   **毛玻璃卡片设计**：
    *   通用卡片：使用 `.glass` 类名。
    *   高亮/强调卡片：使用 `.glass-strong` 配合 `.glow-border`。
*   **色彩状态管理**：
    *   Cyan (`--cyan`): 用于常规递进动作、基建动作等。
    *   Green (`--green`): 用于创意进阶、优质数据、成功指标。
    *   Red/Orange (`--red` / `--orange`): 用于系统拦截、痛点、尾部客户数据。
    *   Gold (`--gold`): 用于核心破解动作、最高效打法。
*   **动效管理**：给进入视角的元素添加 `.reveal` 和 `.reveal-delay-X`（X为1-5），增强讲解时的逐步呈现感。

### Step 3: 数据脱敏与合规审查 (Planner + Deployer)
在交付客户或外部公开前，**必须**执行严格的脱敏流程：
*   **大盘数据**：具体的客户数（如 `17,898`、`14,206`）统一替换为模糊量级（如 **`1万+`**）。
*   **各梯队数据**：各分层的具体商家数量（如 `102家`、`902家`、`13,202家`）应**直接删除**，仅保留百分比占比。
*   **客户案例**：如无授权，必须隐去客户真实简称，替换为“行业头部客户A”、“大健康赛道某商家”等。
*   **金额数据**：日耗等数据保留规模区间（如 `3K~5W`），避免透露过于精确的 BI 绝对值。

### Step 4: 文件覆盖与上线部署 (Deployer)
1. **本地调试**：使用 Live Server 等本地环境（如 `http://localhost:5500`）预览确认排版与动效。
2. **版本锁定**：将完成修改的源文件（如 `xxx_v7.html`）保存在根目录，作为存档。
3. **覆盖上线**：
   *   复制当前定稿的内容：`cp xxx_v7.html deploy/index.html`。
   *   确保 `deploy` 目录包含最新的静态资源（如果有新增的图片或 CSS）。
   *   将 `deploy` 目录推送至 GitHub 仓库的主分支或 `gh-pages` 分支。
4. **验证**：访问线上 GitHub Pages 链接，确保多端展示正常。

## 三、 🤖 附：给 AI 助手的 Prompt 模板
当团队需要让 AI（如 CodeBuddy）快速生成新页面时，可以直接复制以下话术：

> “请基于 `食饮投放力诊断与提升_客户培训_v6.html` 的 UI 风格和 HTML 结构，帮我生成一份关于【XXX主题】的客户诊断报告。
> 
> 内容大纲如下：
> 1. 左侧（占比4列）：XXX痛点描述，使用橙色/红色预警样式；
> 2. 右侧（占比8列）：对应3个优化抓手，使用 Cyan 和 Green 的毛玻璃卡片（.glass）展示。
> 
> 要求：保持原有的 `.reveal` 进场动效，标题使用 `.gradient-text` 渐变效果，注意所有具体客户数据直接用‘万+’脱敏。”
