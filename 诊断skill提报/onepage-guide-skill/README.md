# One Page 制作指南 · 技能包

> 一个完全自包含的 One Page（可视化单页）制作技能包。  
> 复制整个文件夹即可使用，不依赖任何外部文件。

---

## 这是什么？

这个技能包帮你快速制作**面向客户传播的可视化单页文档（One Page）**，适用于：

- 投放动态 / 规则变更的客户周知
- 平台能力 / 功能推广指引
- 扶持政策 / 激励方案通知
- 任何需要"截图发给客户"的信息传达

产出物是一个 **HTML 单页**，可在浏览器中直接编辑文字，一键导出 **4K 高清 PNG 图片**。

## 使用方式

### 方式一：AI 辅助制作（推荐）

如果你使用 CodeBuddy 或其他 AI 编程助手：

1. **安装技能**：将整个 `onepage-guide-skill/` 文件夹复制到你项目的 `.codebuddy/skills/` 目录下
2. **触发 AI**：在对话中告诉 AI 你要做什么，例如：

```
请帮我制作一个面向客户的 One Page，内容是本周的投放动态周知。
以下是需要传达的信息：
1. xxx
2. xxx
3. xxx
```

AI 会自动参考 `SKILL.md` 中的规范来制作 One Page。

**其他触发提示词示例**：
- "基于以下内部通知，制作面向客户的可视化单页"
- "帮我把这些信息做成一个可以截图发给客户的 One Page"
- "按照 One Page 制作规范，生成客户周知 HTML"

### 方式二：手动制作

不使用 AI 助手时，也可以手动制作：

1. **复制模板**：复制 `template.html` 并重命名（如 `客户周知_2026.05.01.html`）
2. **填写内容**：用编辑器打开，按照 `<!-- 替换为... -->` 注释提示填入内容
3. **浏览器预览**：双击 HTML 文件在浏览器中打开预览效果
4. **编辑微调**：页面文字可直接在浏览器中点击编辑
5. **导出截图**：点击右下角「📸 导出 4K 图片」按钮，自动下载 PNG

> 手动制作时，建议先阅读 `SKILL.md` 中的"二、信息架构规范"和"四、视觉设计规范"两章，了解标准组件的用法。

## 标准组件速查

模板中预置了以下标准组件，覆盖常见的 One Page 信息类型：

| 组件 | CSS 类名 | 用途 |
|:---|:---|:---|
| 模块卡片 | `.mod` `.mod-h` `.mod-b` | 信息分组（渐变头部 + 白底内容区） |
| 变化标签 | `.lb .lb-change` | 📢 平台变化 / 规则说明 |
| 行动标签 | `.lb .lb-action` | ✅ 行动建议 |
| P0 行动项 | `.ar` + `.d-p0` | 最高优先级操作建议 |
| P1 行动项 | `.ar` + `.d-b` | 次要优先级操作建议 |
| KPI 卡片 | `.kpi-row` `.km` | 数据指标展示 |
| 扶持卡片 | `.bonus` | 🎁 限时扶持 / 激励信息 |
| 商家 Todo | `.todo` | 📌 客户需要执行的操作链路 |
| 链接按钮 | `.lk` `.lk-b` `.lk-o` | 参考文档 / 操作指南入口 |
| 公式块 | `.formula` | 概念定义 / 公式解释 |
| 两列布局 | `.row2` | 并排展示两个模块 |

## 截图导出注意事项

导出工具基于 html2canvas，有以下 CSS 限制：

| 不要使用 | 替代方案 |
|:---|:---|
| `backdrop-filter: blur()` | `background: rgba(255,255,255,0.95)` |
| `mix-blend-mode` | `opacity` |
| 复杂 `clip-path` | `border-radius` 或 SVG |
| `background-attachment: fixed` | `scroll` |
| `conic-gradient` | `linear-gradient` |

确保页面宽度 ≥ 1200px，关键数值列加 `white-space: nowrap`。

## 常见问题

**Q：截图按钮不出现？**  
A：确保 `editable-snapshot.js` 和 HTML 文件在同一目录下，且 HTML 中的引用路径是 `<script src="editable-snapshot.js"></script>`。

**Q：截图中部分样式丢失？**  
A：检查是否使用了 html2canvas 不支持的 CSS 属性（见上方表格）。

**Q：如何修改主色调？**  
A：搜索 `#2563eb`（蓝色系）替换为你需要的颜色即可。模块头部渐变色在 `.mod-h` 的 `style` 属性中单独设置。

**Q：如何添加更多模块？**  
A：复制模板中的 `.mod` 块，修改内容即可。两列布局用 `.row2` 包裹两个 `.mod`。
